# 冗余代码删除与迭代计划模板

## 优先级

- [ ] **P0**：立即删除（安全风险、显著成本、阻塞其他工作）
- [ ] **P1**：本迭代内删除
- [ ] **P2**：待办 / 下次迭代

---

## 立即可安全删除

### 条目：[名称/描述]

| 字段 | 详情 |
|------|------|
| **位置** | `path/to/file.py:行号` |
| **删除理由** | 为何应该删除 |
| **证据** | 无引用（未使用）、特性开关已关闭、已废弃的 API |
| **影响** | 无 / 低 - 无活跃消费方 |
| **删除步骤** | 1. 删除代码 2. 删除测试 3. 删除配置 |
| **验证方式** | 运行测试，确认无运行时错误，监控日志 |

---

## 延迟删除（需要计划）

### 条目：[名称/描述]

| 字段 | 详情 |
|------|------|
| **位置** | `path/to/file.py:行号` |
| **延迟原因** | 有活跃消费方、需要迁移、需要干系人确认 |
| **前置条件** | 特性开关关闭超过 2 周，遥测显示使用量为 0 |
| **破坏性变更** | 列出所有 API/契约变更 |
| **迁移计划** | 消费方迁移步骤 |
| **时间线** | 目标日期或迭代 |
| **负责人** | 负责的人员/团队 |
| **验证指标** | 确认可安全删除的指标（错误率、使用量统计） |
| **回滚方案** | 发现问题时如何恢复 |

---

## 删除前检查清单

- [ ] 在代码库中搜索所有引用（`grep`、`rg`）
- [ ] 检查是否存在动态/反射调用
- [ ] 确认无外部消费方（API、SDK、文档）
- [ ] 审查特性开关遥测数据（如适用）
- [ ] 更新/删除相关测试
- [ ] 更新文档
- [ ] 通知团队（如果是共享代码）
