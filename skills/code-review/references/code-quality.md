# 代码质量清单

## 错误处理

### 需标记的反模式

- **吞掉的异常**：空的 catch 块或只有日志的 catch
  ```python
  try: ...
  except Exception: pass  # 静默失败

  try: ...
  except Exception as e: print(e)  # 记录后忽略
  ```
- **过于宽泛的 catch**：捕获 `Exception`/`BaseException` 基类而非具体类型
- **错误信息泄露**：向用户暴露堆栈跟踪或内部细节
- **缺失错误处理**：可能失败的操作（I/O、网络、解析）没有 try-except
- **异步错误处理**：未处理的 Promise 拒绝、缺少 `.catch()`、无错误边界

### 最佳实践检查

- [ ] 在适当的边界捕获错误
- [ ] 错误信息对用户友好（不暴露内部细节）
- [ ] 错误日志包含足够的调试上下文
- [ ] 异步错误被正确传播或处理
- [ ] 可恢复错误定义了降级行为
- [ ] 严重错误触发告警/监控

- "调用方能知道出错了吗？"
- "这个错误有足够的上下文来调试吗？"

## 性能与缓存

### CPU 密集操作

- **热路径中的耗时操作**：循环中的正则编译、JSON 解析、加密运算
- **阻塞主线程**：同步 I/O、无 worker/async 的重计算
- **不必要的重复计算**：相同计算执行多次
- **缺少记忆化**：相同输入的纯函数被反复调用

### 数据库与 I/O

- **N+1 查询**：循环中对每条记录单独查询而非批量
  ```python
  # 差：N+1
  for id in ids:
      user = db.query(User).filter(User.id == id).first()

  # 好：批量查询
  users = db.query(User).filter(User.id.in_(ids)).all()
  ```
- **缺少索引**：对未建索引的列进行查询
- **过度查询**：`SELECT *` 而只需要少数几列
- **无分页**：将整个数据集加载到内存

### 缓存问题

- **耗时操作缺少缓存**：重复的 API 调用、数据库查询、计算
- **缓存无 TTL**：过期数据被无限期提供
- **缓存无失效策略**：数据已更新但缓存未清除
- **缓存键冲突**：键的唯一性不足
- **用户特定数据全局缓存**：安全/隐私问题

### 内存

- **无界集合**：无限增长的数组/字典
- **大对象持有引用**：阻止垃圾回收
- **循环中字符串拼接**：应使用 join 代替
- **完整加载大文件**：应使用流式处理

### 自问
- "这个操作的时间复杂度是多少？"
- "数据量扩大 10 倍/100 倍时表现如何？"
- "这个结果可以缓存吗？应该缓存吗？"
- "能批量处理而非逐条处理吗？"

## 边界条件

### Null/None 处理

- **缺少空值检查**：访问可能为 None 的对象的属性
- **真值/假值混淆**：`if value:` 在 `0` 或 `""` 是合法值时失效
- **可选链过度使用**：`a?.b?.c?.d` 掩盖了结构性问题
- **None 与未定义不一致**：混用而无明确约定

### 空集合

- **未处理空数组**：代码假设数组有元素
- **空对象边界情况**：对空对象使用 `for...in` 或 `keys()`
- **首/末元素访问**：`arr[0]` 或 `arr[-1]` 未检查长度

### 数值边界

- **除以零**：除法前缺少检查
- **整数溢出**：超出安全整数范围的大数
- **浮点数比较**：使用 `==` 而非 epsilon 比较
- **负值**：不应为负的索引或计数
- **差一错误**：循环边界、数组切片、分页

### 字符串边界

- **空字符串**：未作为边界情况处理
- **纯空白字符串**：通过真值检查但实际为空
- **超长字符串**：无长度限制导致内存/显示问题
- **Unicode 边界情况**：emoji、RTL 文本、组合字符

### 需标记的常见模式

```python
# 危险：无空值检查
name = user.profile.name

# 危险：无检查的数组访问
first = items[0]

# 危险：无检查的除法
avg = total / count

# 危险：真值检查排除了合法值
if value:  # 对 0、""、False 失效
    ...
```
