本文件定义后端 API 设计文档的标准结构和写作指引。生成时必须严格遵循此模板，并与项目已有的 `doc/平台后端API设计文档/` 格式保持一致。

保存为 `doc/平台后端API设计文档/{编号}-{模块名称}API.md`，编号与需求文档保持一致。

## 接口设计规范

### 请求方法映射

| 操作 | HTTP 方法 | URL 模式 |
|------|-----------|----------|
| 获取列表 | GET | /{resources} |
| 获取详情 | GET | /{resources}/{id} |
| 创建 | POST | /{resources} |
| 更新 | PUT | /{resources}/{id} |
| 删除 | DELETE | /{resources}/{id} |
| 批量删除 | DELETE | /{resources}/batch |
| 特殊操作 | POST | /{resources}/{id}/{action} |

### URL 命名规范

- 资源名使用英文复数形式：`/projects`、`/bugs`、`/test-cases`
- 多单词用中划线连接：`/test-cases`、`/api-documents`
- 嵌套资源最多两层：`/projects/{project_id}/modules`
- 操作类接口用动词：`/projects/{id}/archive`、`/bugs/{id}/assign`

---

## 参数规范

### 列表查询参数（所有列表接口必须支持）

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| page | int | 否 | 页码，默认1 |
| page_size | int | 否 | 每页数量，默认20 |
| keyword | string | 否 | 搜索关键词 |
| order_by | string | 否 | 排序字段 |
| order | string | 否 | 排序方向：asc/desc |

### 路径参数格式

| 参数 | 类型 | 说明 |
|------|------|------|
| {resource}_id | int | 资源ID |

### 请求体格式

- 创建/更新接口使用 JSON Body
- 参数说明表固定 4 列：`| 字段 | 类型 | 必填 | 说明 |`
- 类型使用小写：string, int, boolean, date, array, object

## 响应格式规范

### 列表响应

```json
{
  "code": 200,
  "message": "success",
  "data": {
    "total": 50,
    "page": 1,
    "page_size": 20,
    "items": [...]
  }
}
```

### 详情响应

```json
{
  "code": 200,
  "message": "success",
  "data": {
    "id": 1,
    "...": "..."
  }
}
```

### 创建/更新响应

```json
{
  "code": 200,
  "message": "创建成功",
  "data": {
    "id": 1,
    "name": "..."
  }
}
```

### 删除响应

```json
{
  "code": 200,
  "message": "删除成功",
  "data": null
}
```

## 接口文档写作指引

### 每个接口必须包含

1. **接口地址**：`METHOD /path` 格式
2. **权限要求**：标注该接口所需的角色或权限（如 `管理员`、`登录用户均可`、`公开接口`）
3. **请求参数**：Query 参数用表格，Body 参数用 JSON 示例 + 参数说明表
4. **响应示例**：完整的 JSON 响应，字段值使用有意义的示例数据（非占位符）
5. **业务错误**：列出该接口特有的错误场景（通用错误码在 2.4 节已覆盖，无需重复）

### 接口分组

- 按子模块分组（如：项目管理接口、模块管理接口、成员管理接口）
- 每组内按 CRUD 顺序排列：列表 → 详情 → 创建 → 更新 → 删除 → 特殊操作

### 字段一致性

- 响应字段必须与数据库设计中的字段对应
- 敏感字段（如 password_hash）不得出现在响应中
- 关联对象使用嵌套对象展示（如 `created_by: { id, username, real_name }`）
- 时间字段统一使用 ISO 8601 格式：`2025-01-01T00:00:00Z`

### 从需求文档推导接口

- 需求文档中每个"操作列表"中的操作 → 对应一个 API 接口
- 需求文档中的"输入项/配置项" → 对应创建/更新接口的请求参数
- 需求文档中的"输出结果" → 对应详情接口的响应字段
- 需求文档中的"业务规则" → 对应接口的验证逻辑和错误场景
